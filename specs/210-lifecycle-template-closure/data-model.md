# Phase 1 Data Model: Close the Lifecycle-Agnostic Template Epic

This feature's "entities" are documents and coordination records, not runtime types. Each is defined by its
required fields, validation rules, and (where applicable) state.

## Epic Acceptance Record  (FR-001..006)

The single consolidated artifact that proves "emit Spec Kit only when asked" against the published package.
Lives at `specs/210-lifecycle-template-closure/readiness/epic-acceptance.md`, generated by the harness.

| Field | Description | Validation |
|---|---|---|
| `validated_package` | Exact package id + version under test | MUST equal `FS.GG.UI.Template 0.1.51-preview.1` (the user-confirmed pin) |
| `tag_anchor` | Nearest git tag + note on template-tag gap | MUST cite `fs-gg-ui/v0.1.51-preview.1` and state "no dedicated template tag at 0.1.51; follow-up: tag `fs-gg-ui-template/v0.1.51-preview.1`" |
| `provenance` | `live` or `verdict-core` | Close/don't-close conclusion VALID only when `live` (Constitution V) |
| `profiles` | The supported profile set | MUST be exactly `[app, headless-scene, governed, sample-pack]` |
| `gated_set_definition` | The gated lifecycle file set (restated) | MUST match Feature 204's set: `.specify/`, generated constitution, `.agents/`, `.claude/`, generated `AGENTS.md`/`CLAUDE.md` |
| `result.spec_kit` | Per-profile: full lifecycle surface emitted | All 4 profiles → gated set **present** |
| `result.sdd` | Per-profile: lifecycle surface suppressed, product intact & buildable | All 4 → gated set **absent**, product **present**; `app`-profile **builds** (spot-check `dotnet build` exit 0, or `environment-limited` disclosed) |
| `result.none` | Per-profile: surface + orchestrator expectation absent, product intact & buildable | All 4 → gated set **absent**, **no orchestrator marker**, product **present**; `app`-profile **builds** (spot-check exit 0, or `environment-limited` disclosed) |
| `buildability` | Build spot-check outcome for `sdd`/`none` `app`-profile | `pass` (both exit 0) \| `environment-limited` (toolchain absent, disclosed); a silent skip is invalid (SC-007) |
| `byte_identical` | Default(`spec-kit`) vs pre-lifecycle baseline | All 4 → presence+content diff = **none**; baseline restated |
| `byte_identical_definition` | Baseline, profile set, presence-vs-content scope | MUST state baseline source (204/206), all 4 profiles, "both presence and content compared" |
| `reproduction` | Exact commands to re-run from the pin | MUST be runnable verbatim and reproduce the verdict |
| `conclusion` | Rendering-side close / don't-close statement | One explicit decision; references the remainder (see Closure State) |

**State**: `draft` (verdict-core synthesized) → `verified` (live run, all results green) → cited at closure.
A `verified` record with any misclassified gated file, non-zero byte diff, or a **failed** build spot-check
(exit ≠ 0 — distinct from a disclosed `environment-limited`) is **invalid** and blocks close.

## Lifecycle Guidance  (FR-007..009)

Consumer-facing material added to `.template.package/README.md`.

| Field | Description | Validation |
|---|---|---|
| `decision_tree` | Maps the 3 consumer scenarios to values | MUST cover governed→`spec-kit`, SDD-composed→`sdd`, standalone→`none` |
| `per_value_include_exclude` | What each value emits / suppresses | MUST list, for all 3 values, included and excluded surfaces |
| `standalone_none_statement` | Explicit no-governance/no-orchestrator note | MUST state none is attached AND none is expected |
| `migration_note` | Pre-lifecycle → lifecycle-aware upgrade | MUST state "select default `spec-kit` to reproduce prior output" |

**Validation rule (SC-004)**: a reader unfamiliar with the template can pick the correct value for all three
scenarios from the guidance alone.

## Cross-Repo Remainder Item  (FR-010)

A tracked request/decision for work owned by another repo, referenced from the closure record.

| Field | Description | Validation |
|---|---|---|
| `owning_repo` | Repo responsible | e.g. `FS-GG/FS.GG.SDD` |
| `kind` | `request` or `decision` | — |
| `tracking_url` | Link to the existing/created issue | MUST be reused if already open (no duplicates — edge case) |
| `blocks` | What full closure it blocks | — |
| `referenced_from_record` | Linked in the closure record | MUST be true |

**Instances** (expected):
1. SDD scaffold-path obligations → **reuse** `FS-GG/FS.GG.SDD#1` (request).
2. Constitution-ownership for `lifecycle=sdd` → tracked **decision** item (reuse if an issue exists; else create).

## Closure State  (FR-011)

The board-visible status of the epic on the `FS-GG` Projects v2 "Coordination" board.

| Field | Description | Validation |
|---|---|---|
| `rendering_status` | Rendering-side completion state | Set to **complete** only when the acceptance record is `verified` |
| `remaining_items` | List of remainder items with owners | Each entry attributed to its owning repo with a tracking link |
| `epic_fully_done` | Distinguishes Rendering-done from epic-fully-done | MUST be `false` while any remainder item is open |

**Invariant (SC-005/006)**: every SDD-owned remaining item has **exactly one** tracked request/decision
referenced from the closure record — no untracked blockers, no duplicates — and the board reflects this.
