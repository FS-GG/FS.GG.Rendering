Rationale for V:
  The template pins EVERY FS.GG.UI.* package to a single $(FsSkiaUiVersion), so the feed must be
  COHERENT (one version for all packages). A bare `dotnet pack` (no -p:Version) honours each src
  project's own <Version> and produces a MIXED set (Scene 0.1.37, Controls 0.1.46, SkiaViewer 0.1.47,
  Canvas/Symbology 0.1.0, …) that a single pin cannot resolve. Packing with -p:Version=V overrides all
  projects to one coherent V (the dev-repack mechanism; matches the historical single-version pin, e.g.
  the prior 0.1.129-preview.1).
  V chosen = 0.1.48-preview.1: one above the current max per-project version (0.1.47-preview.1, SkiaViewer),
  follows the repo 0.1.N-preview.1 scheme, and is FRESH (absent from the global NuGet cache, which already
  holds 0.1.37/0.1.46/0.1.47 — packing at an existing version risks serving stale cached bits and would
  invalidate the drift signal). No src/**/*.fsproj <Version> nor Directory.Build.props <Version> was
  edited (research Decision 3: version bumps are owned by the merge process, not this task).
