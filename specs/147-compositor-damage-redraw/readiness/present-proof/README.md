# Present Proof Evidence

This directory contains Feature147 present-path proof artifacts for the active host profile.

## Artifact Schema

- `proof.md`: markdown proof record generated by `compositor-present-proof`.
- Host profile fields: profile id, backend, renderer, present mode, framebuffer, display environment, and proof algorithm version.
- Verdict values: `passed`, `failed`, or `environment-limited`.
- Evidence artifacts: paths to screenshots, readback data, or deterministic harness notes used by the proof.
- Diagnostics: reasons why proof is stale, mismatched, failed, synthetic, or environment-limited.

## Acceptance Rules

- A proof enables scissor readiness only when it is fresh, passed, and matches the active host profile.
- Missing, stale, host-mismatched, failed, synthetic, or environment-limited proof keeps the compositor in full-redraw fallback.
- Deterministic harness output may document the contract, but it does not replace live host timing or presentation evidence for a shipped performance claim.
