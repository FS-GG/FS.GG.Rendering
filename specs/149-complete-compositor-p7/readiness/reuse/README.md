# Feature 149 Reuse Artifacts

## Schema

- `reuse.md`: corpus decisions for stable, placement-only, mixed-change, no-change, content-changing, churning, no-benefit, failed-parity, and same-seed scenarios.
- `demotions.md`: demotion reasons for content, resource, provider, host-profile, parity, churn, and no-benefit changes.
- `work-reduction.md`: repeated-work reduction and measured overhead summary.

## Acceptance

Reuse requires deterministic content/placement identities, old/new movement damage, parity-clean output, and measured benefit before a ready claim.
