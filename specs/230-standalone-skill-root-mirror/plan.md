# Implementation Plan: standalone spec-kit product mirrors the skill union into all three agent roots

**Branch**: `230-standalone-skill-root-mirror` | **Date**: 2026-07-01 | **Spec**: [spec.md](./spec.md)

## Summary

Corrects Feature 229's over-confinement. **ADR-0011 §1** requires all three agent-skill roots (`.agents/`, `.claude/`, `.codex/`) to carry the byte-identical skill union; **§2** makes `fsgg-sdd` the mirror authority for **orchestrated** scaffolds. Feature 229 confined the provider to `.agents/skills/` under **every** lifecycle — correct for `sdd`/`none` (orchestrator fans out; keeps Templates#47 unblocked) but wrong for the **standalone `spec-kit`** lane, which has no orchestrator, so its product ended up with skills only in `.agents/skills/`.

**Fix**: in the standalone `spec-kit` lane the template self-fans-out the skill union into all three roots; `sdd`/`none` stay `.agents/`-only. Mechanism: for each of the 12 sources that populate `.agents/skills/` under `spec-kit` (the base repo-root blanket, the 9 `template/product-skills/` sources, samples, feedback), add a `.claude/skills/` twin and a `.codex/skills/` twin, each `spec-kit`-gated. Per-skill twins (not a blanket copy) are required because the repo-root `.agents/skills/` copies differ byte-wise from the canonical `template/product-skills/` versions (verified). Tier 2 content/config + gate rework; a template version bump ships it.

## Phase 0 / 1 notes
