# Implementation Plan: Promote Token, Policy & Resolver Surface to Public — Workstream F, Phase F5

**Branch**: `130-promote-token-surface` | **Date**: 2026-06-16 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/130-promote-token-surface/spec.md`

## Summary

F5 is the deliberate **public-surface promotion** that F1–F4 each deferred. It takes two proven-internal
capabilities and makes them a public, documented, baseline-gated contract:

1. The **Ant-derived token taxonomy** — `DesignTokensExt` (F1/126): seed → map → alias → component layers
   plus `Space`/`Density`/`Type`/`Elevation`. Today `module internal DesignTokensExt` with **no `.fsi`**,
   GENERATED by `scripts/generate-design-tokens.fsx`.
2. The **central style resolver + intent-policy seam** — `StyleResolver` (F4/129): the total path
   `theme + kind + intent + state(s) → ResolvedStyle`, the overridable `IntentPolicy` record, `neutralPolicy`,
   `baseStyleFor`, `resolve`, and `resolveDefault`. Today `module internal StyleResolver` with no `.fsi`.

Promotion follows **Principle II** (visibility lives in `.fsi`, not `.fs` access modifiers): we **remove the
`internal` modifier** from each `.fs` and **add a curated `.fsi`** declaring the public surface — for the
generated taxonomy the generator is extended to emit a paired **generated `.fsi`** in lock-step with the `.fs`.
The change is **surface-only**: token *values* are byte-identical (regenerated from the same DTCG source), the
public `Theme` record shape is unchanged, and rendered output is byte-identical under the default neutral policy.

The third candidate — the F2/127 **color-validation policy** (`wcag`/`ant`) — is **deferred** (decision recorded,
not promoted): `FS.GG.UI.Color` carries **no surface baseline** at all (it is absent from the refresh table), the
`--design-system wcag|ant` template parameter (F3/128) already exposes the policy *choice* to consumers, and no
current consumer needs the engine itself public. Promoting unguarded `Color` internals would contradict the
"deliberate, baseline-gated" discipline this phase exists to uphold.

Both CI drift gates move in the **same change**: the per-package public-surface baseline for
`FS.GG.UI.DesignSystem` (the only package whose surface grows) and the design-token-drift generator (`--check`).
A decision record under `docs/product/decisions/` (`0004`) names every promoted symbol, the deferred color
policy, the rationale, and the stability/reversibility commitment.

This is a **Tier 1 (contracted)** change — the first in Workstream F — so it carries the full chain: spec, plan,
`.fsi` additions, surface-baseline regeneration, public-path test evidence, and docs/decision-record updates.

## Technical Context

**Language/Version**: F# on .NET `net10.0`, single solution `FS.GG.Rendering.slnx`, `TreatWarningsAsErrors=true`
(all code must be warning-clean).

**Primary Dependencies**: None added (FR-011). Promotion reuses existing in-repo assemblies only —
`FS.GG.UI.DesignSystem` (`Theme`, `ResolvedStyle`, `Style.resolve`, `StyleClass`, `VisualState`, the generated
token modules) and `FS.GG.UI.Scene` (`Color`/`Colors`). No JSON parser, web, React/DOM, or icon-font dependency
enters any product/test assembly; `System.Text.Json` stays confined to the generator *script* (never compiled
into a product/test assembly), exactly as F1 established.

**Storage**: N/A (token values are compile-time literals; the DTCG `design-tokens.tokens.json` is the source of
truth read only by the generator script).

**Testing**: xUnit-style headless deterministic tier (no GL context required). New public-path tests land in
`tests/Controls.Tests/Feature130PublicSurfaceTests.fs` and MUST reach the promoted symbols through the **public
API only** (they do not rely on any `InternalsVisibleTo` grant). Existing Feature126/Feature129 suites stay green
unchanged. Full suite via `FS.GG.Rendering.slnx`.

**Target Platform**: Linux (dev/CI) headless for all F5 gates; the promoted surface is platform-neutral.

**Project Type**: F# UI rendering framework (multi-project single solution). **No new project** (this is the key
divergence from D1, which added packages — F5 grows the surface of one *existing* package).

**Performance Goals**: Neutral. Promotion changes visibility only — no new allocation, no new runtime path, no
change to at-rest/animation/layout/cache behaviour. `buttonGeom` still calls `StyleResolver.resolveDefault`
exactly as F4 wired it; the call is now to a public binding rather than an internal one (identical IL).

**Constraints**:
- **Zero token-value delta** and **no `Theme` record shape change** (FR-004, SC-003) — promotion is visibility-only.
- **Byte-identical rendered output** under the default neutral policy across the full intent × visual-state
  cross-product (FR-004, SC-003).
- The **only** per-package surface-baseline that changes is `FS.GG.UI.DesignSystem.txt`, and every added row
  corresponds one-to-one to a promoted symbol named in the decision record (FR-005, SC-002, SC-007).
- The **design-token-drift generator** (`--check`) stays green after regeneration (committed `.fs`/`.fsi` ==
  regenerated output).
- Visibility declared in `.fsi`, **never** via `.fs` access modifiers (Principle II, FR-007).
- New tests reach the surface through the public API only (FR-008); test pass/skip counts change only by the
  additive F5 tests (FR-012, SC-006).

**Scale/Scope**: Two `.fs` files lose their `internal` modifier; two `.fsi` files are added (one hand-curated for
`StyleResolver`, one **generated** for `DesignTokensExt`); the generator script gains `.fsi` emission + extends
`--check` to both files; the `DesignSystem.fsproj` compile list gains the two `.fsi` entries (and the now-redundant
`InternalsVisibleTo` grants are removed); the `FS.GG.UI.DesignSystem` surface baseline is regenerated; one new test
file; one decision record; docs touch-ups (`module-map.md`). No control code changes; no other package changes.

**Structure Decision**: Grow the surface of **one existing package** (`FS.GG.UI.DesignSystem`) rather than add a
project (the divergence from D1). The taxonomy keeps the **generated-`.fs` + paired-`.fsi`** shape that
`DesignTokens` already uses, except the `.fsi` is *generated* too (justified by its ~130-leaf size — hand-curation
would drift from the generator; see R3). `StyleResolver`'s `.fsi` is **hand-curated** (small, stable, semantic).
Both `.fsi` files compile **before** their `.fs` in `DesignSystem.fsproj`. Because the promoted symbols are no
longer internal, the F1/F4 `InternalsVisibleTo` grants are removed (Controls and Controls.Tests now consume the
public surface) — IVT is invisible to the surface gate, so this is baseline-neutral. No other package re-exports
these symbols, so `FS.GG.UI.DesignSystem.txt` is the sole baseline that changes.
