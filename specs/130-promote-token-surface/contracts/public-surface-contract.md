# Contract — The Promoted Public Surface

This is the exact public surface F5 adds to `FS.GG.UI.DesignSystem`. The `.fsi` files below are the contract;
the implementation must match them and nothing beyond them becomes public.

## `StyleResolver.fsi` (hand-curated, NEW)

```fsharp
namespace FS.GG.UI.DesignSystem

open FS.GG.UI.Scene

/// Front half of style resolution: theme + control-kind + semantic-intent + visual-state(s) -> ResolvedStyle.
/// Supplies the kind's structural base (under an overridable intent policy), then composes the 093 back-half
/// Style.resolve for the class+state overlay. Total + deterministic; the default path is intent-neutral.
module StyleResolver =

    /// The overridable (theme, lowered-intent-string, structural base) -> adjusted base seam.
    /// `neutralPolicy` keeps it identity; a divergent policy (e.g. "danger" -> theme.Danger) drives intent
    /// divergence with no control edits.
    type IntentPolicy =
        { ApplyIntent: Theme -> string -> ResolvedStyle -> ResolvedStyle }

    /// The kind's structural base style. Total over kind: "icon-button" -> accent outline; any other kind ->
    /// filled accent (a defined, visible fallback — never empty, transparent-only, or an exception).
    val baseStyleFor: theme: Theme -> kind: string -> ResolvedStyle

    /// The default, intent-agnostic policy: returns the structural base unchanged.
    val neutralPolicy: IntentPolicy

    /// The single front-half resolution path. Total + deterministic.
    val resolve:
        policy: IntentPolicy ->
        theme: Theme ->
        kind: string ->
        intent: string ->
        classes: StyleClass list ->
        state: VisualState ->
            ResolvedStyle

    /// The neutral path control render code calls — intent threaded but ignored, byte-identical to the
    /// pre-promotion internal call.
    val resolveDefault:
        theme: Theme ->
        kind: string ->
        intent: string ->
        classes: StyleClass list ->
        state: VisualState ->
            ResolvedStyle
```

**Constraints**: exactly these five members are public; no helper leaks. `internal` is removed from
`StyleResolver.fs`; its body is otherwise unchanged.

## `DesignTokensExt.fsi` (GENERATED, NEW)

Generated by `scripts/generate-design-tokens.fsx` in lock-step with `DesignTokensExt.fs`. Shape (abbreviated —
the generator emits a `val name : type` for every leaf the `.fs` emits as a `let`):

Each nested module carries a generated `///` doc comment describing the layer (FR-010, module-granularity — per-leaf
doc comments are not emitted):

```fsharp
namespace FS.GG.UI.DesignSystem

open FS.GG.UI.Scene

/// The Ant-derived design-token taxonomy: seed -> map -> alias -> component layers plus spacing, density,
/// type scale, and elevation. Generated from the DTCG source; values are byte-identical to the flat primitives.
module DesignTokensExt =
    /// Seed layer: the primitive brand/semantic colors and base scalar units the map layer derives from.
    module Seed =
        val colorPrimary: Color
        val colorSuccess: Color
        // ... colorWarning/Error/Info, colorTextBase/BgBase : Color
        val fontSize: float
        // ... lineHeight, borderRadius, controlHeight, sizeUnit, sizeStep, motionUnit : float
    module Map =
        module Light = // colorPrimaryHover/Active/Bg, colorErrorBg, colorBorder, colorFillSecondary,
                       // colorBgContainer/Elevated/Layout, colorText/Secondary/Disabled : Color
        module Dark  = // (same leaf names)
    module Alias =
        module Light = // textDefault/Secondary, surfaceCanvas/Container/Elevated, borderDefault,
                       // itemHoverBg/SelectedBg, focusRing, feedbackErrorText/WarningText : Color
        module Dark  = // (same leaf names)
    module Component = // Button/Input/Table/Tabs/Menu/... sub-modules of Color tokens
    module Space     = // xs/sm/md/lg/xl : float
    module Density   = // comfortable/middle/compact : float
    module Type =
        module Display = // fontSize/lineHeight : float
        // Section/Title/Body/Small
    module Elevation = // none/low/medium/high : string
```

**Constraints**: the `.fsi` declares the full taxonomy and nothing else; it is byte-locked to the `.fs` (generator
`--check` green); `internal` is removed from the generated `module` declaration. Token *values* are unchanged. Each
nested module carries a generated `///` doc comment (FR-010, module-granularity); per-leaf doc comments are not
emitted.

## Compile order (`DesignSystem.fsproj`)

Each `.fsi` is added immediately before its `.fs`, after `Style.fs`:

```
DesignTokens.fsi / DesignTokens.fs            (unchanged)
DesignTokensExt.fsi  (NEW)  / DesignTokensExt.fs   (regenerated, internal removed)
Style.fsi / Style.fs                          (unchanged)
StyleResolver.fsi  (NEW)  / StyleResolver.fs  (internal removed)
```

The two `InternalsVisibleTo` items (`Controls.Tests`, `FS.GG.UI.Controls`) added by F1/F4 are **removed** — the
promoted symbols are now public, so the grants are redundant. (IVT is not public surface, so this is baseline-
neutral; guarded by a green full-suite build.)

## Out of contract (not promoted)

- `FS.GG.UI.Color.ColorPolicy` — stays internal (deferred; recorded in `0004`).
- The flat `DesignTokens` module — unchanged; **not** unified with `DesignTokensExt` in F5 (deferred).
- The public `Theme` record — shape unchanged.
